# サンプルデータ生成機能の実装プロンプト

以下の機能を実装してください。

## 目的
このアプリを他のユーザーに配布する際、ユーザーが自分のコンテキスト用 `.md` ファイルをまだ持っていないケースに対応する。「サンプルデータを作成」ボタンを提供し、サンプル `.md` ファイル群を自動生成できるようにする。

## 要件

### 1. サンプルデータの内容（10ファイル）

以下のバリエーションで構成する。各ファイルは実際のナレッジメモらしい構成にする（見出し、箇条書き、コード例、日付などを含む）。`config.yaml` の `language` 設定に応じて **日本語または英語** で生成する。ファイル名は言語に依存しない英語ベース。

| # | カテゴリ | ファイル名 | テーマ例 |
|---|---|---|---|
| 1 | 技術学習メモ | `python-basics.md` | Python基礎（型、リスト内包表記、デコレータ等） |
| 2 | 技術学習メモ | `git-workflow.md` | Git操作（ブランチ戦略、rebase、cherry-pick等） |
| 3 | 技術学習メモ | `docker-intro.md` | Docker入門（Dockerfile、コンテナ、ネットワーク等） |
| 4 | 読書ノート | `book-clean-code.md` | 技術書の要約（命名規則、関数設計、テスト等） |
| 5 | 読書ノート | `book-team-management.md` | ビジネス書の要約（チーム運営、1on1、フィードバック等） |
| 6 | TIL | `til-2025-06.md` | 6月のTIL集（5〜7項目の短い学び） |
| 7 | TIL | `til-2025-07.md` | 7月のTIL集（5〜7項目の短い学び） |
| 8 | 技術調査レポート | `comparison-ci-cd-tools.md` | CI/CDツール比較（GitHub Actions vs Azure Pipelines等）。Pros/Cons表を含む |
| 9 | 業務手順書 | `runbook-deployment.md` | デプロイ手順（ステップバイステップ形式、チェックリスト付き） |
| 10 | 振り返り | `retrospective-2025-q2.md` | 四半期KPT（Keep/Problem/Try）、目標の進捗 |

### 2. サンプルデータ生成モジュール（新規: `app/sample_data.py`）
- `generate_sample_data(target_dir: Path, language: str) -> list[Path]` 関数を作成する
- すべてのサンプルコンテンツは `_SAMPLES` リスト/辞書に `ja` / `en` ペアで定義する
- 既存ファイルがある場合は上書きしない（スキップしてログ出力）
- 各ファイルは十分な量の本文を含める（クイズ・ブリーフィング生成のテスト素材として機能するため）

### 3. UI: セットアップウィザード（`app/setup_wizard.py`）
- フォルダ指定ステップに「サンプルデータを作成」ボタンを追加する
- ボタン押下時の動作:
  1. フォルダ選択ダイアログを表示（サンプルデータの保存先を選択）
  2. 選択されたフォルダにサンプル `.md` ファイルを生成
  3. そのフォルダパスを `input_folders` に自動設定する
- 生成完了後、生成したファイル数を表示する

### 4. UI: 設定画面（`app/settings_ui.py`）
- フォルダ指定セクションの近くに「サンプルデータを作成」ボタンを追加する
- 動作はウィザードと同様（フォルダ選択→生成→パス追加）

### 5. i18n対応
- ボタンラベル、成功メッセージ、エラーメッセージなど、新しい UI 文字列はすべて `app/i18n.py` の `_STRINGS` に `ja` / `en` 両方のキーを追加する
- キーの命名規則: `sample.` プレフィックスを使う（例: `sample.button_label`, `sample.success_message`）

### 6. テスト
- `tests/test_sample_data.py` を作成し、以下をテストする:
  - 日本語・英語それぞれでファイルが正しく生成されること（10ファイル）
  - 既存ファイルがスキップされること
  - 生成されたファイルが有効な Markdown であること（見出し `#` を含む等）
  - 不正な language 指定時のフォールバック動作

### 制約
- 既存の `app/i18n.py` の `t()` 関数と `_STRINGS` 辞書の構造に従うこと
- 既存テスト（154件）が引き続きパスすること
